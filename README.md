## About Me

我是一名偏后端和工程效率方向的软件工程师，主要做业务系统、自动化平台和高并发服务。相比追逐新概念，我更关心一件事：怎么把复杂流程拆清楚，把系统边界定稳，再用代码和工具把重复工作交给机器。

最近我把更多时间放在 Agent 和 AI Coding 上，重点不是让 AI 多写几行代码，而是让它参与需求澄清、任务拆解、代码实现、测试验证和代码审查。Claude Code、Codex 这类工具真正有价值的地方，是进入工程闭环，而不是停留在一次性的代码生成。

- 常做的事：Python / PHP / Go 后端服务、自动化脚本、任务调度、容器化部署
- 关心的问题：接口边界、幂等、状态流转、并发处理、可观测性和交付验证
- 正在探索：Agent 工作流、工具调用编排、上下文管理、AI 辅助重构
- 个人偏好：先把问题讲清楚，再做最小可验证实现，少写没有业务收益的抽象

## What I'm Exploring

- 把 AI Coding 接入真实开发流程，而不是只拿来生成片段代码
- 从业务流程里抽出稳定的任务链路，用自动化减少人工重复操作
- 用更清晰的状态机、队列、缓存和幂等设计支撑后端系统稳定运行
